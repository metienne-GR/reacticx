## Animations are not working

Make sure `react-native-reanimated` is installed and its Babel plugin is added to your config:

When using React Native Community CLI, you also need to manually add the react-native-worklets/plugin plugin to your babel.config.js.

```js title="babel.config.js"
module.exports = {
  presets: ["babel-preset-expo"],
  plugins: ["react-native-reanimated/plugin"],
};
```

The Reanimated plugin **must be listed last** in the `plugins` array. After adding it, clear the Metro cache:

```bash
npx expo start --clear
```

## Skia components crash or show a blank screen

React Native Skia requires native modules. Make sure you've installed it with Expo:

```bash
npx expo install @shopify/react-native-skia
```

If you're using a bare workflow, run `npx pod-install` (iOS) after installing. Skia **does not work in Expo Go** — use a [development build](https://docs.expo.dev/develop/development-builds/introduction/) instead.

## Gestures don't respond

Wrap your root layout with `GestureHandlerRootView`:

```tsx title="app/_layout.tsx"
import { GestureHandlerRootView } from "react-native-gesture-handler";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <GestureHandlerRootView style={{ flex: 1 }}>
      {children}
    </GestureHandlerRootView>
  );
}
```

## The component import fails with "module not found"

Check that the file was copied into your project by the CLI. If it exists, confirm your `tsconfig.json` has a paths alias for `@/`:

```json title="tsconfig.json"
{
  "extends": "expo/tsconfig.base",
  "compilerOptions": {
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

## Haptics throw an error on Android emulator

`expo-haptics` is a no-op on emulators that don't support vibration. Wrap calls in a try/catch or check `Platform.OS` if you need to guard against it:

```tsx
import * as Haptics from "expo-haptics";
import { Platform } from "react-native";

if (Platform.OS !== "web") {
  Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light).catch(() => {});
}
```

## Components look unstyled or use wrong fonts

Reacticx components use your app's default styles. If something looks off, make sure your `app.json` / `app.config.ts` loads any custom fonts before the component renders. Use `expo-font` or `useFonts` from `expo-font` to preload fonts in your root layout.
