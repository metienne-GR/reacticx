Once a Reacticx component is installed, you can import it and use it in your application like any other React Native component. Components are added directly into your codebase — not hidden inside a library — so you have full control to edit, extend, or restyle them to match your app.

---

## How components are added

When you run the Reacticx CLI, the component source file is copied into your project (typically under `components/ui/`). You own the code from that point on.

```bash
npx reacticx add <component-name>
```

After the command runs, you'll see the new file in your project and can import it right away.

---

## Basic Example

```tsx title="app/index.tsx"
import Glow from "@/components/base/glow";
import { View } from "react-native";

export default function Screen() {
  return (
    <View style={{ flex: 1, justifyContent: "center", alignItems: "center" }}>
      <Glow size={2} style="withoutEasing">
        <Text
          style={[
            styles.text,
            {
              fontFamily: fontLoaded ? "HelveticaNowDisplay" : undefined,
            },
          ]}
        >
          Generate
        </Text>
      </Glow>
    </View>
  );
}
```

---

## Customizing a component

Because the file lives in your project, you can edit it freely. For example, to change the default glow color:

```tsx title="components/ui/glow-button.tsx" {3}
// Find the style or prop that controls the glow and update it
const GLOW_COLOR = "#a855f7"; // purple instead of the default
```

You don't need to fork anything or wait for a library update.

---

## Peer dependencies

Most components rely on one or more of the following packages. Install only what you need:

```bash
npx expo install react-native-reanimated react-native-gesture-handler
npx expo install @shopify/react-native-skia
npx expo install expo-haptics expo-blur
```

Each component's docs page lists the exact peer dependencies it requires.

---

## TypeScript support

All components ship with full TypeScript types. Props are documented inline so you get autocomplete and type-checking out of the box — no extra `@types` packages needed.
